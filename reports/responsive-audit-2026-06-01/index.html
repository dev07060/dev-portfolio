Responsive UX Audit

dev-portfolio 모바일 반응형 분석

2026-06-01 기준 로컬 Next.js 사이트를 실제 브라우저에서 1440x900, 768x1024, 390x844, 360x740 뷰포트로 확인했다. 앱 코드는 변경하지 않았고, 아래 내용은 화면 캡처와 컴포넌트 코드 라인을 함께 본 진단 결과다.

대상: http://127.0.0.1:3000 / cwd: /Users/dev_bh/Desktop/works/portfolio/dev-portfolio

좋은 점 0px 390px/360px 모바일 뷰포트에서 문서 가로 overflow는 감지되지 않았다.
모바일 비용 6.4k px 390x844 영문 기준 전체 페이지 높이. 카드와 헤더가 길어 스캔 비용이 크다.
핵심 리스크 Modal 전역 언어 스위처와 모달/닫기 버튼 레이어가 모바일에서 충돌한다.

요약 판단

기본 grid collapse, 가로 overflow 방지, modal scroll lock, 이미지 preload, 프레젠테이션 swipe 지원은 이미 잘 들어가 있다. 다만 모바일에서는 "작아졌을 뿐 같은 정보 구조"에 가깝고, 고정 요소와 큰 preview 영역이 실제 콘텐츠 탐색을 방해한다.

우선순위 이슈

P1 / 모바일 모달

언어 스위처가 모달 닫기 버튼과 겹친다

LanguageSwitcher.tsx:9의 전역 fixed top-4 right-4 z-50가 ProjectModal.tsx:23의 z-40 모달보다 위에 있다. 닫기 버튼도 ProjectModal.tsx:38에서 z-50라서 모바일에서는 두 컨트롤이 같은 모서리에서 경쟁한다.

  • 개선 방향: 모달/프레젠테이션이 열릴 때 언어 스위처를 숨기거나, 모달 레이어를 명확히 더 위로 올린다.
  • 추가 개선: 닫기 버튼은 스크롤 컨테이너 안의 absolute가 아니라 viewport 기준 fixed 또는 컨테이너 내부 sticky로 둔다.
  • Trade-off: 모달 안에서 언어 전환은 덜 편해지지만, 닫기/탐색의 안정성이 더 중요하다.
P1 / 모바일 모달

모달을 스크롤하면 닫기 버튼이 사라진다

모달 컨테이너는 ProjectModal.tsx:34에서 overflow-y-auto이고, 닫기 버튼은 같은 컨테이너 내부 absolute다. 모바일에서 상세 설명을 읽다가 빠져나갈 명확한 컨트롤이 없어지는 흐름이 생긴다.

  • 개선 방향: 모바일 모달 헤더를 별도 sticky row로 만들고, close/language/status를 한 곳에 둔다.
  • Trade-off: 상단 44-56px 정도의 고정 영역이 생기지만, 긴 프로젝트 설명에서 사용성이 좋아진다.
P1 / 프레젠테이션

웹 프로젝트 프레젠테이션의 세로 배분이 모바일에 맞지 않는다

DeviceFrame.tsx:232에서 scrollable 웹 프레임은 height: 70vh를 사용한다. 여기에 PresentationOverlay.tsx:87의 네비게이션 바와 PresentationOverlay.tsx:136의 caption이 더해져 작은 모바일 화면에서는 설명 영역이 하단에 붙거나 일부가 답답하게 보인다.

  • 개선 방향: 프레임 높이를 calc(100dvh - caption - nav - safe-area) 기반으로 제한한다.
  • 웹 screenshot은 모바일에서는 세로 mock browser보다 full-width preview + 내부 스크롤로 단순화하는 편이 낫다.
  • Trade-off: 데스크톱 같은 "큰 mockup" 감성은 줄지만, 모바일에서는 설명과 조작 버튼이 안정적으로 남는다.
P2 / 첫 화면

모바일 첫 화면의 정보 밀도가 낮다

ProfileHeader.tsx:14의 pt-20 pb-12, 큰 avatar, 전체 tech stack 노출 때문에 390px 폭에서 Open Source 섹션이 약 946px 지점, 360px 폭에서는 약 1060px 지점부터 시작한다. 첫 화면에서 "프로젝트를 볼 수 있는 사이트"라는 신호가 늦다.

  • 개선 방향: 모바일에서는 avatar 크기/상단 padding을 줄이고, tech stack은 핵심 6-8개만 우선 노출한다.
  • 대안: Languages/Mobile/Web/Backend를 accordion 또는 compact two-column chip group으로 바꾼다.
  • Trade-off: 자기소개 감성은 조금 줄지만, 포트폴리오 탐색 속도는 좋아진다.
P2 / 한국어 모바일

bullet 정렬과 한국어 줄바꿈이 어색하다

ProfileHeader.tsx:49, ProfileHeader.tsx:70의 items-center 때문에 2줄 bullet의 점이 첫 줄이 아니라 문장 중앙에 붙는다. 한국어에서는 "배포" 같은 단어가 줄 끝에서 쪼개져 읽힘도 낮아진다.

  • 개선 방향: bullet row는 items-start, dot은 mt-2로 조정한다.
  • 한국어 텍스트에는 제한적으로 break-keep 또는 word-break: keep-all을 적용한다.
  • Trade-off: keep-all은 일부 긴 문장에서 줄 수가 늘어날 수 있어 모바일 문구도 같이 다듬는 편이 좋다.
P2 / 섹션 헤더

모바일 section header가 desktop flex를 그대로 따른다

OpenSourceBanner.tsx:80와 ProjectGrid.tsx:17은 모두 flex items-baseline justify-between이다. 영문 Open Source Packages는 두 줄로 떨어지는데 pub.dev 메타는 우측에 남아 시선 흐름이 갈라진다.

  • 개선 방향: 모바일에서는 flex-col items-start gap-2로 전환하고 meta tracking을 줄인다.
  • Trade-off: header 높이는 조금 늘지만, 제목과 보조 정보의 관계가 더 명확해진다.
P2 / 프로젝트 카드

카드가 모바일에서 너무 길고, 실제 작업물이 잘 드러나지 않는다

ProjectCard.tsx:38의 176px 썸네일은 아이콘 중심이라 프로젝트 내용을 빠르게 비교하기 어렵다. 설명은 ProjectCard.tsx:67에서 2줄 clamp라 중요한 맥락이 잘린다. 8개 프로젝트를 전부 보려면 모바일에서 긴 스크롤이 필요하다.

  • 개선 방향: 모바일 카드 썸네일 높이를 132-152px로 줄이고, 대표 screenshot crop 또는 핵심 metric을 넣는다.
  • 개선 방향: 첫 2-3개 프로젝트는 featured density를 높이고 나머지는 compact list로 전환하는 것도 가능하다.
  • Trade-off: 현재 카드의 여백감은 줄지만, 포트폴리오의 증거성이 좋아진다.
P3 / 터치와 접근성

클릭 가능한 카드가 semantic button/link가 아니다

ProjectCard.tsx:32는 div onClick이다. 모바일 사용자에게는 문제가 덜 보여도 키보드 접근성과 screen reader 측면에서 카드가 명령인지 명확하지 않다. 모달의 preview 클릭도 DeviceFrame.tsx:35의 시각 badge에 의존한다.

  • 개선 방향: 카드 root를 button으로 바꾸거나 role="button", tabIndex, keyboard handler를 추가한다.
  • Trade-off: button reset 스타일이 필요하지만, interaction contract가 명확해진다.

현재 잘 되어 있는 부분

  • 390px/360px 폭에서 문서 전체의 가로 overflow는 발생하지 않았다.
  • 프로젝트 grid는 모바일에서 1열로 자연스럽게 접힌다.
  • 모달 오픈 시 background scroll lock과 이미지 preload가 구현되어 있다.
  • 프레젠테이션 모드에 swipe navigation이 있어 터치 사용성의 기반은 있다.

추천 진행 순서

단계 목표 수정 후보
1 모바일 모달 안전성 확보 언어 스위처 overlay policy, sticky/fixed close, modal z-index 정리
2 첫 화면 모바일 밀도 개선 ProfileHeader spacing, tech stack compact layout, bullet alignment
3 프로젝트 탐색 속도 개선 ProjectCard mobile thumbnail/card density, 대표 screenshot 도입
4 프레젠테이션 모드 정리 dvh 기반 frame sizing, caption/nav reserve, scroll indicator 위치 보정

캡처 증거

검증 메모: 문서 가로 overflow는 390x844/360x740에서 0px로 측정되었다. 캡처 중 좌하단의 검은 원형 N 표시는 Next.js dev overlay로 production UI 이슈로 보지 않았다.
